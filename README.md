Запуск: make up

в infra/ca лежит корневой серт
в infra/kafka лежат серты для брокеров
в infra/services лежат серты для сервисов

В internal/*.go лежит рабочий код c продюсерами и консьюмерами из 2 модуля. в cmd/main.go происходит запуск сервиса

Проверить работу можно, выполнив "docker logs kafka-practicum-service-1" и "docker logs kafka-practicum-service-2" (для каждой из реплик) и увидев, что сообщения отправляются, увидеть ID продьюсеров (рандомно назначается при старте сервиса), увидеть сообщения, увидеть размер батчей. Все в логах

В сервисе 2 сообщения должны только отправляться, но не читаться (запрет через ACL)
